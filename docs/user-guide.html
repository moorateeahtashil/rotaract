2. Roles & Access Control

Each user holds one or more roles, ranked by privilege (lower number = more access). Effective access follows the user's highest role.

LvlRoleAccess
0Super AdminEverything, incl. managing other admins and deleting users.
1AdminFull Admin Portal + Member Portal.
2Board Member (+ legacy: President, Secretary…)Member Portal + elevated rights on some content.
3MemberFull Member Portal; listed in the directory.
4Prospective MemberMember Portal (dashboard, profile, events, attendance, announcements, directory, resources). Not yet in the directory; sees their membership progress.
5Normal / PublicPublic website only.

Access is enforced at three layers that share one definition: middleware (route guard), server layout guards, and database row-level security. An admin can never assign a role higher than their own; only a Super Admin creates Super Admins.

3. Getting Started

Create an account

  1. Go to /signup (or click Join / Sign Up).
  2. Enter name, email, password → submit.
  3. You're signed in as a prospective member and taken to the Member Portal.

Log in / reset password

Log in at /login; use Forgot password to receive a reset link. Invited by an admin? Click Accept Invitation in the email to set a password and activate your account.

4. Guide for Prospective Members

You get the Member Portal while your membership is being earned/approved:

  • Dashboard — a QR check-in card, announcements, upcoming events, and your membership progress (turns green as you meet each requirement).
  • My Profile — photo, occupation, contact details.
  • My Events — upcoming club events.
  • Attendance — check in at events (camera QR or code).
  • Announcements, Directory, Resources.
Attend the required events (see §8). When you've met all requirements your card turns green and an admin promotes you to full Member — at which point you also appear in the directory.

5. Guide for Members

Dashboard

QR check-in card, current announcements, upcoming events, quick stats.

Check in to an event

On the dashboard (or the Attendance page) tap Scan QR and point at the event's QR code, or type the event code. Done instantly.

My Profile

Update details/avatar; toggle directory visibility.

My Events

Upcoming events with date, time, location.

Announcements / Directory / Resources

Read club announcements; browse and search fellow members; download shared documents (public or member-only).

6. Guide for Administrators

The Admin Portal (/admin) groups tools in the left sidebar. Most follow the same pattern: a list with Add / Edit / Delete and a dialog form.

People

Members

The roster. Add sends an email invite. Approve a prospect to make them a Member. Edit roles. Delete users.

Applications

Public Join-form submissions. Approve invites & onboards them as a Member (with email); Reject with notes.

Attendance

Mark Attendance — pick an event, tick who attended (members & prospects). By Person — everyone's attendance; prospects show requirement progress + a green Ready badge and a Promote button. Requirements — set how many of each event type a prospect must attend.

Board Members & Committees

Board roster with positions; committees with a chair.

Program

Events & Event Types

Create events with an image, date/time, location, capacity, status, and an event type. Each event has a QR button to generate its attendance code. Event Types are managed separately — six core types (Project, Fellowship, Professional Development, Club Meeting, Club Assembly, Board Meeting) are locked and cannot be deleted.

Projects

Service projects. Creating a project also creates a linked Project-type event so it's attendable and counts toward requirements.

News & Announcements

Blog posts (publish toggle); member announcements (priority, expiry, publish).

Media & Resources

Gallery

Albums + multi-image upload.

Resources

Documents with access level (public / member-only / board-only).

FAQs

Q&A with category, order, visibility → shown on the public FAQ page.

Sponsors

The sponsoring Rotary club(s) shown on the public Sponsors page.

Content

Site Settings (name, tagline, hero banner, logo, meeting info, social, footer, contact) · Rotary Highlights · Legal Pages.

Insights & System

Analytics (live counts) · Users & Roles (change any user's role) · Audit Log.

7. Event Attendance & QR Check-in

Admin — generate the code

  1. Admin → Events → click the QR button on an event.
  2. A unique code (24-hour expiry) is created and linked to that event; a scannable QR + text code appears.
  3. Display the QR (or code) at the venue. Generating a new code deactivates the old one.

Member / Prospective — check in

  1. On the Dashboard (or Member → Attendance) tap Scan QR and point at the code — or type it in the box.
  2. Attendance is recorded against that event instantly.
Ended events are closed. If an event has finished (status completed/cancelled, or past its end), check-in is refused. Camera scanning uses the browser's built-in reader; manual entry always works as a fallback.

Admin — manual adjustments

Admin → Attendance → Mark Attendance: pick the event and tick anyone present or absent.

8. Membership Requirements & Promotion

A prospective member becomes a full member by participating. Requirements are configurable per event type.

Set the requirements (admin)

Admin → Attendance → Requirements: for each event type (Club Meeting, Fellowship, Project, etc.) set how many a prospect must attend, plus an optional "project lead roles" count. Set 0 to ignore a type.

How progress is tracked

  • Every event has an event type. A prospect's attendance is grouped by type and compared to the requirements.
  • Prospects see this on their dashboard; admins see it under Attendance → By Person.
  • When all requirements are met → a green "All requirements met" / "Ready" indicator.

Promotion

An admin clicks Promote (on Attendance → By Person, or Approve on Members/Applications). This grants the Member role, creates the member record, and adds them to the directory.

9. Troubleshooting & FAQ

A prospect can't see the portal

Confirm they're signed in with the prospective_member role (Admin → Users & Roles). Prospects are allowed into the portal by design.

"This event has ended" on check-in

Expected — attendance closes when the event finishes. Admins can still set attendance manually under Attendance → Mark.

An attendance code doesn't work

Codes expire after 24h and only one is active per event. Generate a fresh one from the event's QR button.

I approved someone but they're not a member

Use Promote (Attendance → By Person) or Approve (Members). That assigns the Member role and creates the record server-side.

An admin save "doesn't stick"

Ensure the database migrations (through 022) are applied — they add the admin write policies for sponsors, attendance, FAQs, and requirements.

Images not showing on the live site

Storage buckets are public and serve correctly; if images don't render it's usually a stale deployment — redeploy the site.
